surrogateModels#

This package contains an implementation of the hierarchical Kriging multi-fidelity surrogate model by Han and Görtz (2012) DOI: 10.2514/1.J051354

A wrapper around the Gaussian process regression models provided by scikit-learn.

class optimizationTools.surrogateModels.hierarchicalKriging(lf_x, lf_resp, hf_x, hf_resp, scale_hf_input=True, kernel='squared_exponential', lf_kernel='Matern')[source]#

Bases: object

A hierarchical Kriging surrogate model for multifidelity optimization.

This class implements the hierarchical Kriging model by Han and Görtz (2012) (DOI: 10.2514/1.J051354). The general idea is that the low fidelity points are described using a standard Kriging model. The high fidelity points are then described with the low fidelity kriging as a trend function and an additional stationary random process Z

\[Y(x) = \beta_0 \hat{y}_\text{lf}(x) + Z(x)\]
Parameters:

  • lf_x (np.ndarray, shape (n_lf_samples, n_design_variables)) – low fidelity sampling points

  • lf_resp (np.ndarray, shape (n_lf_samples, 1)) – low fidelity model responses corresponding to the sampling points

  • hf_x (np.ndarray, shape (n_hf_samples, n_design_variables)) – high fidelity sampling points

  • hf_resp (np.ndarray, shape (n_hf_samples, 1)) – high fidelity model responses corresponding to the sampling points

  • scale_hf_input (bool) – whether or not the high fidelity input should be scaled to a unit hypercube centered at origin. This may improve the convergence because of better conditioning of the two-point correlation matrix.

  • kernel (str, optional) – specify the kernel to use for estimating the two-point correlation. Must be one of {‘squared_exponential’, ‘cubic_spline’}. Default is ‘squared_exponential’.

  • lf_kernel (str, optional) – kernel to use for fitting the low-fidelity gaussian process. Must be one of {‘RBF’, ‘Matern’}. Default is ‘Matern’.

Variables:

  • beta0 – hierarchical Kriging parameter beta0

  • theta_opt – hierarchical Kriging parameter theta

Example:

Constructing and using a hierarchical Kriging surrogate model

>>> surrogate = ot.surrogateModels.hierarchicalKriging(lf_x, lf_resp, hf_x, hf_resp)
>>> surrogate.predict(np.array([[0.2], [0.4]]))
array([-0.24874606,  0.11477697])

kernel_exp(x_i, x_j, theta, p_k=2.0)[source]#

Squared exponential kernel for approximating a two point correlation. This is the default Gaussian Process Regression Kernel in Dakota.

\[R(x_i, x_j) = \text{exp} \left( - \sum_k^m \theta |x_{i,k} - x_{j,k}|^{p_k} \right)\]

p_k is assumed to be constant for all k

Parameters:

  • x_i (np.ndarray, shape (1, n_design_variables)) – point in design space

  • x_j (np.ndarray, shape (1, n_design_variables)) – point in the design space

  • theta (float) – parameter of the Kriging model. This is tuned by an optimization algorithm

  • p_k (float, optional) – degree of the exponential kernel (default = 2.0)

Return type:

float

Returns:

estimation of the two-point correlation

neg_log_likelihood(theta)[source]#

Negative logarithmic likelihood function that is minimized.

For obtaining the hierarchical Kriging parameters theta and beta0, the likelihood function is maximized. For numerical reasons, the logarithmic likelihood function is maximized. For this maximization, the DIRECT algorithm is used (following the Kriging implementation in Dakota). As this algorithm only supports minimization, the negative logarithmic likelihood function is calculated here. The mathematical formulation is given by Han and Görtz (2012) (DOI: 10.2514/1.J051354):

\[- \text{ln}[L(\theta)] = n \text{ln}(\sigma^2(\theta)) - \text{ln}(|R(\theta)|)\]

Note that minimization of this function yields an optimal theta and implicitly tunes the other parameter beta0.

Parameters:

theta (float) – parameter of the hierarchical Kriging model.

Return type:

float
